# aibrd — Python Library

Standalone Python CLI and importable library for personal repos. Bring your own API key.

---

## Install

```bash
cd pythonlibrary
pip install -e .
```

Or install dependencies directly:
```bash
pip install -r requirements.txt
```

---

## Configure LLM Provider

Set **one** of these environment variables — aibrd auto-detects which to use:

```bash
# Option 1: Anthropic Claude (recommended)
export ANTHROPIC_API_KEY=sk-ant-...

# Option 2: GitHub Models (free with any GitHub account, no extra key)
export GITHUB_TOKEN=ghp_...

# Option 3: OpenAI
export OPENAI_API_KEY=sk-...
```

Override the default model:
```bash
export AIBRD_MODEL=claude-opus-4-7      # for complex BRDs
export AIBRD_MODEL=gpt-4o-mini          # faster / cheaper
```

Provider priority: `ANTHROPIC_API_KEY` → `GITHUB_TOKEN` → `OPENAI_API_KEY`

## CLI Usage

### Initialize from a BRD

```bash
aibrd init ./docs/my-project-brd.pdf
aibrd init ./docs/requirements.docx
aibrd init ./docs/spec.md
```

Generates `.aibrd/` in the current directory. Commit it to git.

```bash
git add .aibrd/
git commit -m "feat: add aibrd living spec"
```

Force reinitialize:
```bash
aibrd init ./docs/brd-v2.pdf --force
```

### Add a new PO requirement

```bash
aibrd update "Users must be able to reset password via SMS OTP. OTP expires in 60 seconds."
```

Appends to the correct `CONTEXT.md` with new stable IDs.

### Generate test cases

```bash
aibrd tests                    # all modules (or flat)
aibrd tests --module payments  # specific module only
```

### Check a file for coverage gaps

```bash
aibrd gaps src/payments/checkout.py
aibrd gaps src/auth/reset.py --module auth
```

### Generate release notes

```bash
aibrd release v2.3.0
aibrd release v2.3.0 --range main...release/v2.3
```

## Use as a Library

```python
from aibrd.parsers import parse_file
from aibrd.chunker import chunk_brd
from aibrd.extractors.flows import extract_flows
from aibrd.extractors.rules import extract_rules
from aibrd.generators.context_md import generate_context_md
from aibrd.analyzers.gap_detector import detect_gaps, format_gap_report

# Parse BRD
brd = parse_file("requirements.pdf")

# Extract
flows = extract_flows(brd.text)
rules = extract_rules(brd.text)

# Analyse gaps
gaps = detect_gaps(open(".aibrd/CONTEXT.md").read(), open("src/payments.py").read())
print(format_gap_report(gaps))
```

## Environment Variables

| Variable | Purpose |
|---|---|
| `ANTHROPIC_API_KEY` | Use Claude (Anthropic) |
| `GITHUB_TOKEN` | Use GitHub Models API (free) |
| `OPENAI_API_KEY` | Use OpenAI |
| `AIBRD_MODEL` | Override default model |
| `AIBRD_CHUNK_TOKENS` | Max tokens per chunk (default: 6000) |

## Output — `.aibrd/` Folder

Same structure as the VS Code extension:

```
.aibrd/
├── registry.json          # ID counter — never reused
├── CONTEXT.md             # Living spec (flat) or
├── modules/               # per-module CONTEXT.md (large projects)
├── index.md               # Traceability matrix
├── ambiguity-report.md
├── conflict-report.md
├── tests/
│   └── test-cases.md
└── releases/
    └── v1.0.md
```

CONTEXT.md generated by the Python library is **identical in format** to the VS Code extension output — both tools are compatible on the same repo.
